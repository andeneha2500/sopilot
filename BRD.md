## SOPilot — AI-Powered SOP Generator

**Version:** 1.0  
**Type:** Internal POC Prototype  
**Date:** May 2026  
**Status:** Draft

---

## 1. Project Overview

SOPilot is an AI-powered web application that converts plain-English descriptions of business processes into fully structured Standard Operating Procedure (SOP) documents. The tool eliminates the manual effort of writing SOPs by leveraging a large language model to generate consistent, professional, ready-to-use documents in seconds.

---

## 2. Problem Statement

Business teams waste hours writing Standard Operating Procedures manually. The process is inconsistent, often skipped, and hard to maintain. There is no quick way to go from "I know this process" to a professional, structured document.

## 4. Functional Requirements

### FR1 — Process Input
- User must be able to type a free-text description of any business process
- Minimum 10 characters, maximum 1000 characters
- Input field must have a placeholder example to guide the user

### FR2 — Context Selection
- User selects one department from: HR, Finance, Operations, IT, Sales, Other
- User selects complexity level: Simple (1-page SOP) or Detailed (multi-section SOP)

### FR3 — AI SOP Generation
- System sends user input + context + complexity to AI model
- AI returns a structured SOP containing exactly these sections:
  - Title
  - Purpose
  - Scope
  - Roles & Responsibilities
  - Pre-requisites
  - Step-by-Step Procedure
  - Edge Cases & Exceptions
  - Success Metrics
- Generation must complete within 30 seconds

### FR4 — Refinement Loop
- After SOP is generated, user can type a refinement instruction
- Example instructions: "make it more detailed", "add a compliance section"
- System regenerates the SOP with the refinement applied
- This loop can repeat unlimited times

### FR5 — PDF Export
- User can download the generated SOP as a formatted PDF
- PDF must include: title, all sections, generation date, and a "Generated by SOPilot" footer

### FR6 — Copy to Clipboard
- One-click copy of the full SOP text

## 7. Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| Language | Python 3.11 | Fast to build, widely known |
| UI Framework | Streamlit | Zero frontend code, professional look |
| AI Model | Claude API (claude-sonnet-4-20250514) | Best instruction-following for structured docs |
| PDF Export | ReportLab | Pure Python, no dependency headaches |
| Deployment | Local (Docker optional) | POC scope |
| Version Control | GitHub | Portfolio visibility |

---

## 8. Application Architecture

```
User Input (Streamlit UI)
        ↓
Context + Complexity Selection
        ↓
Prompt Builder — constructs structured prompt
        ↓
Claude API Call (Anthropic SDK)
        ↓
SOP Response Parser
        ↓
Streamlit Display + PDF Generator (ReportLab)
        ↓
Download Button / Copy Button
```
